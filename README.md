## 1. Проблема

Текущая чат-система не пригодна для эксплуатации клубной системы:

| Проблема | Влияние |
|----------|---------|
| Все сообщения в одном общем чате | Невозможно вести переписку с конкретным игроком |
| Нет push-уведомлений при свёрнутом приложении | Игроки и менеджеры не узнают о новых сообщениях |
| Нет уведомлений о заявках в клуб | Заявки теряются, игроки уходят |
| Чаты всех клубов смешаны | Путаница при работе с несколькими клубами |
| Нет бейджей с числом непрочитанных | Непонятно, сколько сообщений пропущено |
| Нет поиска/фильтрации игроков | Надо вручную листать members |
| Нет отдельных чатов стафф-стафф | Менеджеры не могут переписываться между собой |

**Вывод Владимира:** "Существование любого приложения происходит с чатом. Чат работает с уведомлениями всегда. Если в WhatsApp не приходят уведомления — нахуй нужен WhatsApp?"

## 2. Типы чатов (архитектура)

### 2.1. Чат "Игрок ↔ Клуб" (Player-to-Club)

```
┌─────────┐          ┌──────────────────────┐
│  Игрок  │ ◄──────► │       КЛУБ           │
│ (1 чат) │          │ (владелец + админы +  │
│         │          │  менеджеры видят всё) │
└─────────┘          └──────────────────────┘
```

- **Со стороны игрока**: один чат "с клубом"
- **Со стороны стаффа**: видят переписку со ВСЕМИ игроками, каждый игрок — отдельный тред
- Все менеджеры/админы/владелец видят одну и ту же переписку
- Ответ от любого стаффера приходит игроку как "от клуба"
- Новое сообщение от игрока — чат поднимается вверх в списке у стаффа

### 2.2. Чат "Стафф ↔ Стафф" (Staff-to-Staff)

```
┌───────────┐          ┌───────────┐
│ Владелец  │ ◄──────► │ Менеджер  │
└───────────┘          └───────────┘
      ▲                      ▲
      │                      │
      ▼                      ▼
┌───────────┐          ┌───────────┐
│   Админ   │ ◄──────► │ Менеджер2 │
└───────────┘          └───────────┘
```

- Личные чаты между членами стаффа клуба
- Отдельный от переписки с игроками

### 2.3. Общий чат клуба (Broadcast)

- Для объявлений на всех участников клуба
- Пишут только стафф, игроки читают
- Остаётся как есть (доработать уведомления)

## 3. Роли и права доступа

| Роль | Видит чаты с игроками | Пишет игрокам | Стафф-чат | Общий чат (write) |
|------|----------------------|---------------|-----------|-------------------|
| Владелец клуба | Все | Да (от имени клуба) | Да | Да |
| Админ | Все | Да (от имени клуба) | Да | Да |
| Менеджер | Все | Да (от имени клуба) | Да | Да (настраиваемо) |
| Игрок | Только свой | Да (пишет клубу) | Нет | Только чтение |

**Сортировка:**
- По дате последнего сообщения (новые сверху) — **по умолчанию**

- По нику игрока (instant search)

**Бейджи:**
- Число непрочитанных сообщений (не просто точка)
- Отдельный бейдж на иконке "Чаты" в навигации

## 6. Push-уведомления

### 6.1. Когда отправляем push:

| Событие | Кому push | Текст |
|---------|-----------|-------|
| Игрок написал в чат | Всем стафф (владелец + админы + менеджеры) | "Ivan_Player: Привет" |
| Стафф написал игроку | Игроку | "KhanGames: Добро пожаловать!" |
| Новая заявка в клуб | Владельцу + админам | "Новая заявка: Ivan_Player" |
| Игрок добавлен в клуб | Игроку | "Вы приняты в клуб KhanGames" |
| Сообщение в общий чат | Всем участникам клуба | "KhanGames: Турнир в 20:00!" |
| Стафф → стафф | Конкретному стафферу | "Юра: Завтра не работаю" |

### 6.2. Технические требования:

- Push работает при свёрнутом приложении (как Telegram)
- Telegram Web App API: используем `web_app_setup_closing_behavior` + Telegram Bot API для push
- **Вариант A**: Через Telegram Bot API — бот отправляет уведомление пользователю
- **Вариант B**: Нативные Web Push (Service Worker) — если поддерживается
- Бейдж на иконке чата обновляется в реальном времени (WebSocket)

## 8. Модель данных (предварительная)

### Сущности:

```
ChatRoom
├── id: UUID
├── club_id: UUID (FK → Club)
├── type: enum ['player_to_club', 'staff_to_staff', 'broadcast']
├── player_id: UUID | null (FK → User, для player_to_club)
├── created_at: timestamp
├── updated_at: timestamp  // для сортировки по активности
└── last_message_preview: string

ChatMessage
├── id: UUID
├── room_id: UUID (FK → ChatRoom)
├── sender_id: UUID (FK → User)
├── sender_role: enum ['player', 'owner', 'admin', 'manager']
├── content: text
├── created_at: timestamp
├── read_by: UUID[]  // кто прочитал
└── is_system: boolean  // системные сообщения (заявка, добавление)

ChatParticipant
├── room_id: UUID (FK → ChatRoom)
├── user_id: UUID (FK → User)
├── role: enum ['player', 'owner', 'admin', 'manager']
├── unread_count: integer
├── last_read_at: timestamp
└── joined_at: timestamp
```

### Создание чатов:

- **player_to_club**: создаётся автоматически при добавлении игрока в клуб
- **staff_to_staff**: создаётся при первом сообщении между стафферами
- **broadcast**: один на клуб, создаётся при создании клуба

## 9. WebSocket события (Socket.IO)

### Клиент → Сервер:

| Событие | Payload | Описание |
|---------|---------|----------|
| `chat:send_message` | `{ room_id, content }` | Отправка сообщения |
| `chat:mark_read` | `{ room_id }` | Пометить чат прочитанным |
| `chat:typing` | `{ room_id }` | Индикатор набора текста |

### Сервер → Клиент:

| Событие | Payload | Описание |
|---------|---------|----------|
| `chat:new_message` | `{ room_id, message }` | Новое сообщение |
| `chat:message_read` | `{ room_id, user_id }` | Сообщения прочитаны |
| `chat:typing` | `{ room_id, user_id }` | Кто-то печатает |
| `chat:room_updated` | `{ room_id, last_message }` | Обновление превью чата |
| `notification:new` | `{ type, data }` | Push-уведомление |

## 10. API Endpoints (REST)

```
GET    /api/v1/clubs/:clubId/chats              — список чатов клуба (для стаффа)
GET    /api/v1/clubs/:clubId/chats/my            — мой чат с клубом (для игрока)
GET    /api/v1/chats/:roomId/messages            — сообщения чата (пагинация)
POST   /api/v1/chats/:roomId/messages            — отправить сообщение
PUT    /api/v1/chats/:roomId/read                — пометить прочитанным
GET    /api/v1/clubs/:clubId/chats/unread-count   — количество непрочитанных
GET    /api/v1/clubs/:clubId/members/search       — поиск игроков (для начала чата)
```

## 11. Сценарии (User Stories)

### US-1: Игрок пишет в клуб
1. Игрок открывает чат с клубом
2. Пишет сообщение
3. Сообщение приходит всем стаффам клуба (WebSocket)
4. У стаффов бейдж обновляется
5. Если приложение свёрнуто — push-уведомление

### US-2: Менеджер отвечает игроку
1. Менеджер видит список чатов, у Ivan_Player — 3 непрочитанных
2. Открывает чат, видит сообщения
3. Пишет ответ (от имени клуба)
4. Игроку приходит сообщение (WebSocket)
5. У игрока бейдж на чате
6. Если свёрнуто — push

### US-3: Новый игрок добавляется в клуб
1. Игрок подаёт заявку
2. Владельцу/админам — push "Новая заявка: ник"
3. Заявка одобрена → автоматически создаётся ChatRoom(type: player_to_club)
4. Игроку — push "Вы приняты в клуб"
5. В списке чатов у стаффа — новый чат вверху

### US-4: Стафф ищет игрока для написания
1. Стафф открывает список чатов
2. Вводит ник в поиск
3. Находит игрока, открывает чат
4. Пишет сообщение

### US-5: Общее объявление
1. Стафф открывает "Общий чат клуба"
2. Пишет "Турнир в 20:00"
3. Все участники клуба получают уведомление

### US-6: Стафф-стафф переписка
1. Менеджер открывает "Стафф-чат"
2. Выбирает владельца
3. Пишет личное сообщение
4. Только владелец видит
