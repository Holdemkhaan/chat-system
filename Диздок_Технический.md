## 1. Проблема

Текущая чат-система не пригодна для эксплуатации клубной системы:

| Проблема | Влияние |
|----------|---------|
| Все сообщения в одном общем чате | Невозможно вести переписку с конкретным игроком |
| Нет push-уведомлений при свёрнутом приложении | Игроки и менеджеры не узнают о новых сообщениях |
| Нет уведомлений о заявках в клуб | Заявки теряются, игроки уходят |
| Чаты всех клубов смешаны | Путаница при работе с несколькими клубами |
| Нет бейджей с числом непрочитанных | Непонятно, сколько сообщений пропущено |
| Нет поиска/фильтрации игроков | Надо вручную листать members |

**Вывод Владимира:** "Существование любого приложения происходит с чатом. Чат работает с уведомлениями всегда. Если в WhatsApp не приходят уведомления — нахуй нужен WhatsApp?"

## 2. Типы чатов (архитектура)

### 2.1. Чат "Игрок ↔ Клуб" (Player-to-Club) — единственный тип чата

```
┌─────────┐          ┌──────────────────────┐
│  Игрок  │ ◄──────► │       КЛУБ           │
│ (1 чат) │          │ (владелец + админы +  │
│         │          │  менеджеры видят всё) │
└─────────┘          └──────────────────────┘
```

- **Со стороны игрока**: один чат "с клубом"
- **Со стороны стаффа**: видят переписку со ВСЕМИ игроками, каждый игрок — отдельный тред
- Все менеджеры/админы/владелец видят одну и ту же переписку
- Ответ от любого стаффера приходит игроку как "от клуба"
- Новое сообщение от игрока — чат поднимается вверх в списке у стаффа
- **Агент = обычный игрок** — имеет такой же чат с клубом, как любой игрок

### 2.2. Функция "Отправить всем" (Send-to-All)

```
┌──────────────────────────────────────┐
│ [Сообщение...]    [☐ Всем] [Отпр. ➤]│
└──────────────────────────────────────┘
```

- Вместо отдельного "общего чата" — **галочка "Отправить всем"** при написании сообщения
- Стафф ставит галочку → сообщение приходит каждому игроку в его индивидуальный чат с клубом
- Игрок видит это сообщение как обычное сообщение от клуба в своём чате
- Доступно только стаффу (владелец, админ, менеджер)

### 2.3. Что УБРАНО (решение Владимира, 03.03.2026)

| Что убрано | Причина |
|-----------|---------|
| Чат "Агент ↔ его игроки" | Агент = обычный игрок. Своим игрокам писать не может. Для связи с игроками использует WhatsApp/Telegram |
| Чат "Агент ↔ Клуб" (отдельный) | Агент общается с клубом через обычный чат игрока |
| Чат "Стафф ↔ Стафф" | Менеджеры общаются между собой в WhatsApp/Telegram. Отдельный чат внутри приложения не нужен |
| Общий чат (Broadcast) | Заменён на галочку "Отправить всем" — проще и не требует отдельного экрана |

## 3. Роли и права доступа

| Роль | Видит чаты с игроками | Пишет игрокам | Подпись для игрока | "Отправить всем" |
|------|----------------------|---------------|-------------------|-----------------|
| Владелец | Все | Да | "[Название клуба]" | Да |
| Админ | Все | Да | "[Название клуба]" | Да |
| Менеджер | Все | Да | "[Название клуба]" | Да |
| **Агент** | **Только свой чат с клубом** | **Нет (пишет только клубу)** | — | **Нет** |
| Игрок | Только свой чат с клубом | Да (пишет клубу) | — | Нет |

### Агент — обычный игрок в чате:

- **Назначается** владельцем или менеджером клуба (роль для бизнес-логики, не для чата)
- **Привязка игрока к агенту**: через инвайт с пометкой на агента ИЛИ ручное назначение владельцем/менеджером
- **В чате**: агент = обычный игрок. Имеет один чат с клубом. Администрация пишет ему как любому другому игроку
- **НЕ может**: писать своим игрокам внутри приложения (для этого использует WhatsApp/Telegram)
- **Привязка игроков к агенту** сохраняется в ClubMember для бизнес-отчётности и управления

```
┌─────────────────────────────────┐
│ 🔍 Поиск по нику...            │
├─────────────────────────────────┤
│ 🔴 3  Ivan_Player              │  <-- 3 непрочитанных
│       "Когда фишки придут?"    │
│       14:32                    │
├─────────────────────────────────┤
│ 🔴 1  Batom_777                │  <-- 1 непрочитанное
│       "Привет"                 │
│       14:15                    │
├─────────────────────────────────┤
│      Alex_Pro                  │  <-- прочитано
│      "Спасибо, всё ок"        │
│      вчера                     │
├─────────────────────────────────┤
│      Агент_Вася                │  <-- агент = обычный игрок в списке

- Нет отдельного "общего чата" и "стафф-чата" — всё в одном списке
- Агент в списке выглядит как обычный игрок

**Сортировка:**
- По дате последнего сообщения (новые сверху) — единственный вариант

**Поиск:**
- По нику игрока (instant search)

**Бейджи:**
- Число непрочитанных сообщений (не просто точка)
- Отдельный бейдж на иконке "Чаты" в навигации

│   Я: Привет, хочу играть       │

```

- Игрок видит все сообщения (и личные, и массовые) как "от клуба" — без разницы
- Агент видит этот же экран — он обычный игрок

## 6. Push-уведомления

### 6.1. Когда отправляем push:

| Событие | Кому push | Текст |
|---------|-----------|-------|
| Игрок написал в чат | Всем стафф (владелец + админы + менеджеры) | "Ivan_Player: Привет" |
| Стафф написал игроку | Игроку | "[Название клуба]: Добро пожаловать!" |
| Стафф отправил "Всем" | Всем игрокам клуба | "[Название клуба]: Турнир в 20:00!" |
| Новая заявка в клуб | Владельцу + админам | "Новая заявка: Ivan_Player" |
| Игрок добавлен в клуб | Игроку | "Вы приняты в клуб [Название клуба]" |

### 6.2. Технические требования:

- Push работает при свёрнутом приложении (как Telegram)
- Telegram Web App API: используем `web_app_setup_closing_behavior` + Telegram Bot API для push
- **Вариант A**: Через Telegram Bot API — бот отправляет уведомление пользователю
- **Вариант B**: Нативные Web Push (Service Worker) — если поддерживается
- Бейдж на иконке чата обновляется в реальном времени (WebSocket)

## 8. Модель данных (предварительная)

### Сущности:

```
ChatRoom
├── id: UUID
├── club_id: UUID (FK → Club)
├── type: enum ['player_to_club']  // единственный тип
├── player_id: UUID (FK → User)
├── created_at: timestamp
├── updated_at: timestamp  // для сортировки по активности
└── last_message_preview: string

ChatMessage
├── id: UUID
├── room_id: UUID (FK → ChatRoom)
├── sender_id: UUID (FK → User)
├── sender_role: enum ['player', 'owner', 'admin', 'manager']
├── content: text
├── created_at: timestamp
├── read_by: UUID[]  // кто прочитал
├── is_system: boolean  // системные сообщения (заявка, добавление)
└── is_broadcast: boolean  // отправлено через "Отправить всем"

ChatParticipant
├── room_id: UUID (FK → ChatRoom)
├── user_id: UUID (FK → User)
├── role: enum ['player', 'owner', 'admin', 'manager']
├── unread_count: integer
├── last_read_at: timestamp
└── joined_at: timestamp

ClubMember (расширение существующей модели)
├── user_id: UUID (FK → User)
├── club_id: UUID (FK → Club)
├── role: enum ['owner', 'admin', 'manager', 'agent', 'player']
├── agent_id: UUID | null (FK → User)  // к какому агенту привязан (для игроков)
└── invited_by_agent: UUID | null       // через чей инвайт пришёл
```

### Создание чатов:

- **player_to_club**: автоматически при добавлении любого участника (игрока или агента)
- Один ChatRoom на каждого участника × клуб

### "Отправить всем" — механизм:

1. Стафф ставит галочку "Всем" → отправляет сообщение
2. Сервер создаёт ChatMessage с `is_broadcast: true` в КАЖДОМ ChatRoom клуба
3. Игрок видит сообщение в своём индивидуальном чате как обычное сообщение от клуба
4. Стафф видит пометку "(отправлено всем)" в интерфейсе

### Привязка игрока к агенту (бизнес-логика, не чат):

1. **Через инвайт**: игрок получает ссылку-приглашение с пометкой на агента → при одобрении заявки `agent_id` проставляется автоматически
2. **Вручную**: владелец/менеджер назначает существующего игрока под агента
3. Привязка влияет на бизнес-отчётность, но **НЕ влияет на чат** — у агента нет доступа к чатам своих игроков

## 9. WebSocket события (Socket.IO)

### Клиент → Сервер:

| Событие | Payload | Описание |
|---------|---------|----------|
| `chat:send_message` | `{ room_id, content, broadcast?: boolean }` | Отправка сообщения (broadcast=true → всем) |
| `chat:mark_read` | `{ room_id }` | Пометить чат прочитанным |
| `chat:typing` | `{ room_id }` | Индикатор набора текста |

### Сервер → Клиент:

| Событие | Payload | Описание |
|---------|---------|----------|
| `chat:new_message` | `{ room_id, message }` | Новое сообщение |
| `chat:message_read` | `{ room_id, user_id }` | Сообщения прочитаны |
| `chat:typing` | `{ room_id, user_id }` | Кто-то печатает |
| `chat:room_updated` | `{ room_id, last_message }` | Обновление превью чата |
| `notification:new` | `{ type, data }` | Push-уведомление |

## 10. API Endpoints (REST)

```
GET    /api/v1/clubs/:clubId/chats                — список чатов клуба (только для стаффа)
GET    /api/v1/clubs/:clubId/chats/my             — мой чат с клубом (для игрока/агента)
GET    /api/v1/chats/:roomId/messages             — сообщения чата (пагинация)
POST   /api/v1/chats/:roomId/messages             — отправить сообщение { content, broadcast?: boolean }
POST   /api/v1/clubs/:clubId/chats/broadcast      — отправить всем игрокам { content }
PUT    /api/v1/chats/:roomId/read                 — пометить прочитанным
GET    /api/v1/clubs/:clubId/chats/unread-count    — количество непрочитанных
GET    /api/v1/clubs/:clubId/members/search        — поиск игроков (для начала чата)
PUT    /api/v1/clubs/:clubId/members/:userId/agent — назначить/сменить агента игроку
POST   /api/v1/clubs/:clubId/invites/agent         — создать инвайт-ссылку с привязкой к агенту
```

## 11. Сценарии (User Stories)

### US-1: Игрок пишет в клуб
1. Игрок открывает чат с клубом
2. Пишет сообщение
3. Сообщение приходит всем стаффам клуба (WebSocket)
4. У стаффов бейдж обновляется
5. Если приложение свёрнуто — push-уведомление

### US-2: Менеджер отвечает игроку
1. Менеджер видит список чатов, у Ivan_Player — 3 непрочитанных
2. Открывает чат, видит сообщения
3. Пишет ответ (от имени клуба)
4. Игроку приходит сообщение (WebSocket)
5. У игрока бейдж на чате
6. Если свёрнуто — push

### US-3: Новый игрок добавляется в клуб
1. Игрок подаёт заявку
2. Владельцу/админам — push "Новая заявка: ник"
3. Заявка одобрена → автоматически создаётся ChatRoom(type: player_to_club)
4. Игроку — push "Вы приняты в клуб"
5. В списке чатов у стаффа — новый чат вверху

### US-4: Стафф ищет игрока для написания
1. Стафф открывает список чатов
2. Вводит ник в поиск
3. Находит игрока, открывает чат
4. Пишет сообщение

### US-5: Рассылка "Отправить всем"
1. Стафф открывает любой чат (или список чатов)
2. Пишет сообщение, ставит галочку "Всем"
3. Сообщение приходит КАЖДОМУ игроку в его индивидуальный чат
4. Все игроки получают push-уведомление
5. Игрок видит это как обычное сообщение от клуба

### US-6: Агент пишет в клуб
1. Агент открывает свой чат с клубом (как обычный игрок)
2. Пишет "Присвой мне Ivan_Player, забыл ссылку"
3. Стафф видит сообщение в списке чатов
4. Отвечает "Ок, привязал"
5. Агент видит ответ от клуба

### US-7: Игрок приходит через инвайт агента
1. Агент генерирует инвайт-ссылку в клуб (с привязкой к себе)
2. Игрок переходит по ссылке, подаёт заявку
3. Владельцу — push "Новая заявка: ник (от агента Вася)"
4. Заявка одобрена → игрок автоматически привязан к агенту (в ClubMember.agent_id)
5. Создаётся обычный ChatRoom(type: player_to_club)

### US-8: Ручная привязка игрока к агенту
1. Агент пишет в клуб: "Ivan_Player — мой игрок, забыл инвайт-ссылку"
2. Владелец/менеджер открывает профиль Ivan_Player
3. Выбирает "Назначить агента" → выбирает агента
4. Игрок привязан к агенту (бизнес-логика, на чат не влияет)
